# readable-doc

知識ベースの .md ファイルを書く・整形するスキル。

## 基本ルール

### 見出しは動詞・場面ベース

「〇〇する」形式。トピックラベルではない。

```
# 悪い例
## 環境変数
## ブランチ操作

# 良い例
## 環境変数を渡す
## ブランチを扱う
```

### `> [!info]` で理由を添える

非自明な選択・設計・手順には必ず「なぜか」を書く。コマンドの羅列は知識にならない。

```markdown
> [!info] なぜ〇〇か
> 理由を1〜3行で。判断基準・トレードオフ・背景を書く。
```

判断不要な自明なセクション（基本コマンドの列挙など）には不要。

### `---` 不使用

水平線はノイズ。セクション区切りは見出しで十分。

### 番号プレフィックス不使用

`## 1. セットアップ` → `## セットアップする`

順序が重要なステップは番号付きリスト（`1.` `2.`）で表現する。

## prompts.md へのエントリ追加

プロンプト集にエントリを追加するときのフォーマット：

### エントリ名

> 先行コンテキスト必須。（事前に情報を共有した後に使う場合のみ）

```
プロンプト本文をそのまま
```

> [!info] なぜ効くか
> このプロンプトが機能する核心的な理由を書く。
> 似たプロンプトとの違いがあれば明示する。

## 整形の手順

既存ファイルをリフォーマットするとき：

1. ファイルを読んで現在の構成を把握する
2. セクションを場面ベースに再分類する（セクション数は変えなくていい）
3. 見出しを動詞形に書き換える
4. `---` と番号プレフィックスを削除する
5. 非自明なセクションに `> [!info]` を追加する（全部につける必要はない）
6. 書き直す（既存ファイルは Edit、全体書き直しのみ Write）
