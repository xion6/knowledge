# AI 活用 個人知識ベース

## 1. CLAUDE.md 設計

プロジェクトルートに置く指示ファイル。チームルール・作業方針・コーディング規約を定義する。
一度書けばセッションをまたいで有効。

### 基本原則

- **シンプル第一** — 影響するコードを最小限にする
- **手を抜かない** — 一時的な修正は避ける
- **影響最小化** — 必要な箇所のみ変更する

### ワークフロー設計（Boris Cherny 氏の知見）

**Planモード**
3ステップ以上の複雑なタスクは必ず計画先行。実装前に承認を得る。

**サブエージェント戦略**
リサーチ・並列分析はメインコンテキスト外に委譲。メインの文脈をクリーンに保つ。

**自己改善ループ**
修正・学習を `tasks/lessons.md` に記録し、次のセッションで継承する。

**検証必須**
完了マーク前に動作を証明すること。「実装した」と「動いた」は別。

### ファイル構成

```
project/
├── CLAUDE.md          # ルール定義
└── tasks/
    ├── todo.md        # 進捗管理
    └── lessons.md     # 学習記録（セッション間継承）
```

### CLAUDE.md テンプレート

```markdown
# プロジェクト名

## コンテキスト
<!-- プロジェクトの目的・技術スタック -->

## 基本原則
- シンプル第一：影響するコードを最小限にする
- 手を抜かない：一時的な修正は避ける
- 影響最小化：必要な箇所のみ変更する

## ワークフロー
- 3ステップ以上のタスクは Plan モードで計画先行
- 完了前に動作確認必須
- 学習・修正は tasks/lessons.md に記録

## コーディング規約
<!-- 言語・フレームワーク固有のルール -->
```

## 2. トークン効率化

### .claudeignore

不要なファイルをコンテキストから除外する。プロジェクトルートに配置。

```
# .claudeignore
node_modules/
.git/
dist/
*.log
*.lock
coverage/
```

### genshijin スキル（日本語出力を約80%削減）

日本語の敬語・クッション言葉・冗長助詞を削除し、技術的内容はそのままに出力を圧縮する。

```
/genshijin          # 通常モード（デフォルト）
/genshijin 丁寧     # ビジネス向け
/genshijin 極限     # 最大圧縮
```

効果の目安：
- CORS説明：1,168 → 217 トークン（82%削減）
- イベントループ説明：1,294 → 244 トークン（81%削減）

### jupytext（Notebook のトークン削減）

`.ipynb` をそのまま渡すとメタデータ・出力結果ごと読み込まれてトークンが膨大になる。
`jupytext` で `.py` に変換してから渡すと最大94%削減できる。

```bash
pip install jupytext
jupytext --to py notebook.ipynb
```

### /clear・セッション管理

コンテキストが肥大化したらリセットする。

```
/clear              # コンテキストをリセット
/compact            # コンテキストを圧縮（リセットより穏やか）
```

長いセッションは途中でリセットして新しいセッションで続けるほうがコスト・品質ともに良い。

### MCP・プラグイン最小化

使わない MCP サーバー・プラグインは無効化する。
起動時にシステムプロンプトへ注入されるためセッション全体のトークンに影響する。

```bash
cat ~/.claude/settings.json | jq '.enabledPlugins'
```

### モデル使い分け

| 用途 | モデル |
|------|--------|
| 複雑な設計・推論 | Opus |
| 通常のコーディング | Sonnet（デフォルト） |
| 単純な変換・確認 | Haiku |

```
/model haiku        # モデルを一時切り替え
```
