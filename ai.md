## CLAUDE.md 設計

プロジェクトルートに置く指示ファイル。セッションをまたいで有効。`/init` で自動生成できる。

### 基本原則

- **シンプル第一** — 影響するコードを最小限にする
- **手を抜かない** — 一時的な修正を避ける
- **影響最小化** — 必要な箇所のみ変更する

### ワークフロー設計（Boris Cherny 氏の知見）

- **Plan モード** — 3ステップ以上は計画先行・承認後に実装
- **サブエージェント** — リサーチ・並列分析はメインコンテキスト外に委譲
- **検証必須** — 完了前に動作証明。「実装した」と「動いた」は別
- **学習継承** — 修正・気づきを `tasks/lessons.md` に記録し次セッションで継承

### テンプレート

```markdown
# プロジェクト名

## コンテキスト
目的・技術スタック・チーム規模など

## 基本原則
- シンプル第一：影響するコードを最小限にする
- 手を抜かない：一時的な修正を避ける

## ワークフロー
- 3ステップ以上は Plan モードで計画先行
- 完了前に動作確認必須
- 学習・修正は tasks/lessons.md に記録

## コーディング規約
言語・フレームワーク固有のルール
```

### ファイル構成

```
project/
├── CLAUDE.md
└── tasks/
    ├── todo.md
    └── lessons.md
```

## トークン効率化

### .claudeignore

不要ファイルをコンテキストから除外する。プロジェクトルートに配置。

```
node_modules/
.git/
dist/
*.log
*.lock
coverage/
```

### Notebook（jupytext）

`.ipynb` をそのまま渡すとメタデータ・出力ごと読み込まれてトークンが膨大になる。`.py` に変換して渡す（最大94%削減）。

```bash
pip install jupytext
jupytext --to py notebook.ipynb
```

### コンテキスト管理

```
/compact    # 圧縮（穏やか）
/clear      # リセット（完全）
```

長いセッションは途中リセット＋新セッション継続がコスト・品質ともに良い。

### MCP・プラグイン最小化

使わない MCP は無効化する。起動時にシステムプロンプトへ注入されるためセッション全体のトークンに影響する。

## Hooks（自動化）

`settings.json` の `hooks` フィールドで定義。イベント駆動でシェルコマンドを実行する。

### イベント種別

| イベント | タイミング |
|---|---|
| `PreToolUse` | ツール実行前（lint、バックアップなど） |
| `PostToolUse` | ツール実行後（フォーマッター、テスト実行など） |
| `Stop` | Claude の応答終了時（デスクトップ通知など） |
| `Notification` | 通知発生時 |

### 設定例

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"完了\" with title \"Claude\"'"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd $CLAUDE_PROJECT_DIR && npx eslint --fix $CLAUDE_TOOL_INPUT_FILE_PATH 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

`matcher` はツール名の正規表現。空文字列は全ツールにマッチ。

設定は `/update-config` スキルを使う（手動編集より安全）。

## メモリシステム

場所: `~/.claude/projects/<project>/memory/`
自動的に会話コンテキストへロードされ、セッションをまたいで有効。

### メモリの種類

| タイプ | 用途 |
|---|---|
| `user` | ユーザーの役割・好み・知識レベル |
| `feedback` | 「こうしてほしい/こうしないで」の指示 |
| `project` | 進行中の作業・決定事項・背景 |
| `reference` | 外部リソースの場所（Linear, Grafana等） |

### 保存のタイミング

- 「覚えておいて」と明示的に言う
- Claude が作業を通じて判断・嗜好を学習したとき（自動）

### 保存されないもの

- コードパターン・アーキテクチャ（コードから読める）
- git 履歴・最近の変更（`git log` が正）
- セッション内の一時的な作業状態

### 構造

```
memory/
├── MEMORY.md          # インデックス（常にコンテキストに読み込まれる）
├── user_role.md
├── feedback_terse.md
└── project_context.md
```

## マルチエージェント戦略

### エージェント種別と使い分け

| タイプ | 用途 | 特徴 |
|---|---|---|
| `Explore` | コードベース探索・ファイル検索 | 高速・読み取り専用 |
| `Plan` | 実装設計・アーキテクチャ検討 | 代替案の比較・トレードオフ評価 |
| `general-purpose` | 複合タスク・リサーチ | 全ツール使用可 |
| `claude-code-guide` | Claude Code/API の質問 | 専門知識あり |

### 並列 vs 直列

- **並列:** 独立したタスク（A ファイルの調査と B ファイルの調査）
- **直列:** 前の結果が次に必要（調査 → 設計 → 実装）

### フォアグラウンド vs バックグラウンド

- **フォアグラウンド（デフォルト）:** 結果が次のステップに必要
- **バックグラウンド:** 長時間タスクを並行して走らせる（完了時に通知）

### メインコンテキストを守る

広範な調査はサブエージェントに委譲する。大量の検索結果がメインの文脈を汚染するのを防ぐ。コンテキストがクリーンなほど推論の質が上がる。
